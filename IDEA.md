GOAL: To create a Claude Code dashboard that have unlimited projects, selected by a local folder, primarily from the /home/%USER%/git folder. The main goal is to mange development with Claude Code for each project separately. Brainstorming, creating specs, planning, developing and reviewing. The dashboard for each project should contain a menu of ideas, specs, plans and currently developing. Currently Developing should contain a cards list of all active session that actually does developing.  

IDEAS: The Ideas view reads ideas (.md files) from a specific project folder, which is selected in project settings. They are displayed as cards, in three columns, with a header and other important information. We have some action buttons on each plan. Each button opens a new session in Claude Code with a specific prompt for continue brainstorming idea, creating a spec session, move to planning, start developing and review. Each action should have their own agent with related skill sets. If the necessary agents or skills doesn't exisits in the project, we propose to start a session that creates those agents and skills. If it's a new project without much content, we can brainstorm an idea for the project that gives valid grounds for creating the necessary agents and skills. When starting a brainstorming session, we should start it with a special prompt. The user should be given a prompt modal in the dashboard where he/she can input the basic main idea which should be the start of the brainstorming. Combined with the special brainstorming prompt from the dashboard system, this should probably give a good starting point for a brainstorming session. After the session is complete we should write the brainstorming result to a file in the project. The folder should be selectable in the project settings.

SPECS: The specs view should display current plans, .md files from a selected folder in project settings. They should be displayed as cards with relevant information, that are clickable and opened in a right sidebar drawer or a modal for "full" information. There should be action buttons for "continue spec" and "create plan"

PLANS: The plans view should display current plans, .md files from a selected folder in project settings. They should be displayed as cards with relevant information, that are clickable and opened in a right sidebar drawer or a modal for "full" information. There should be action buttons for "continue planning" and "start developing"
