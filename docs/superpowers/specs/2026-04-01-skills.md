# Skills (Track E)

**Goal:** Add project-scoped skills ‚Äî markdown files stored on disk that are automatically injected into every session spawned for the project. A two-panel UI lets you view, create, edit, and delete skills.

**Dependencies:** Track C (Agents) ‚Äî the agent detail screen's Skills tab references project skills.

---

## How Skills Work

Skills are markdown files stored at `{project.path}/.skills/` ‚Äî one file per skill, e.g. `.skills/coding-standards.md`. The DB holds an index (name, key, path) but content lives on disk.

At session spawn time, `lib/sessions/spawn.ts` reads all skills for the project, concatenates their markdown content, and appends it to the system prompt alongside any agent instructions. All sessions for the project receive all project skills ‚Äî no per-agent filtering.

`.skills/` is added to `.gitignore` or committed ‚Äî user's choice. The app does not manage git for skill files.

---

## Data Model

### New `skills` table

```sql
CREATE TABLE skills (
  id TEXT PRIMARY KEY,
  project_id TEXT NOT NULL,
  name TEXT NOT NULL,
  key TEXT NOT NULL,
  file_path TEXT NOT NULL,
  created_at TEXT NOT NULL,
  UNIQUE(project_id, key)
)
```

| Column | Notes |
|---|---|
| `key` | URL-safe slug, e.g. `coding-standards`. Unique per project. Generated from name on create. |
| `file_path` | Relative to project root, e.g. `.skills/coding-standards.md` |

Content is read from disk via `fs.readFileSync(path.join(project.path, skill.file_path), 'utf8')` at spawn time and when displaying the editor.

---

## Skills Injection at Spawn

In `lib/session-manager.ts`, before building the system prompt:

```typescript
const skills = getSkillsByProject(db, projectId)
const skillsContent = skills
  .map(s => {
    const content = fs.readFileSync(path.join(project.path, s.file_path), 'utf8')
    return `## Skill: ${s.name}\n\n${content}`
  })
  .join('\n\n---\n\n')
```

Appended to the system prompt after agent instructions (if any). If no skills exist, nothing is appended.

## Skills UI

Route: `/projects/[projectId]/skills`

Two-panel layout:

### Left panel (~240px, fixed)

- Filter input at the top (filters by name, client-side)
- List of skills: each shows `name` (primary) and `key` (muted, monospace)
- Selected skill is highlighted
- **`+` button** at the top-right of the panel ‚Äî opens inline create form

**Inline create form** (appears at top of list when `+` clicked):
- Name input (autofocused)
- Key auto-generated from name (editable)
- Confirm (Enter) creates the skill, opens it in the right panel
- Escape cancels

### Right panel (flex: 1)

**Empty state** (no skill selected): "Select a skill to view or edit it."

**Skill selected:**
- Skill name as heading
- Key shown as muted monospace below name
- Toggle: **View** / **Code** (switches between rendered markdown and raw editor)
- **View mode**: rendered markdown (same styling as existing markdown display in the app)
- **Code mode**: textarea with the raw markdown content, full width, monospace font
- **Save** button (shown in Code mode) ‚Äî writes content to disk via `PATCH /api/skills/[id]` which calls `fs.writeFileSync`
- **Delete** button (always visible, bottom of panel) ‚Äî confirm dialog: "Delete [name]? This will remove the file from disk." On confirm: deletes DB record + file.

## API

### `GET /api/skills?projectId={id}`
Returns all skills for the project (metadata only ‚Äî no content).

### `POST /api/skills`
Body: `{ projectId, name, key? }`
- Generates key from name if not provided: `name.toLowerCase().replace(/[^a-z0-9]+/g, '-')`
- Creates `.skills/` directory if it doesn't exist
- Writes starter template to `.skills/{key}.md`:
  ```markdown
  # {name}

  Describe what this skill does and how agents should use it.
  ```
- Inserts DB record
- Returns skill record

### `GET /api/skills/[id]`
Returns skill metadata + file content (reads from disk).

### `PATCH /api/skills/[id]`
Body: `{ content?: string, name?: string }`
- If `content`: writes to disk
- If `name`: updates DB record (does not rename file)

### `DELETE /api/skills/[id]`
Deletes DB record + file from disk. Returns `{ ok: true }`.

## Agent Detail ‚Äî Skills Tab (Track C)

The Skills tab on the agent detail screen (`/projects/[projectId]/agents/[agentId]`) shows the project's skill list as read-only cards ‚Äî name, key, and a preview of the first line of content. A note: "All project skills are automatically injected into this agent's sessions."

No per-agent skill management in Track E. Per-agent filtering is a future enhancement.
